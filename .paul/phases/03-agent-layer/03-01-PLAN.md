<objective>
## Goal
Implement the LangGraph review workflow: Pydantic schemas, prompt templates, and a four-node StateGraph (explain_groups → flag_review → human_checkpoint → compile_output) that takes a GroupAssignment, generates plain-English group explanations via parallel LLM calls, and routes to a human checkpoint when flags warrant it.

## Purpose
Sits between the matching algorithm (Phase 02) and the operator tool (Phase 05). Gives the Blom founder plain-English explanations of each group so they can approve or override assignments — the "human-in-the-loop" layer that makes the system trustworthy.

## Output
- `src/agent/schemas.py` — Pydantic models: GroupExplanationSchema, OperatorOverride, ReviewedResult
- `src/agent/prompts.py` — SYSTEM_PROMPT and USER_PROMPT_TEMPLATE string constants

    ```python
    from __future__ import annotations
    from typing import Literal
    from pydantic import BaseModel

    class GroupExplanationSchema(BaseModel):
        """Structured LLM output for one group explanation."""
        summary: str              # 2-3 sentences: who this group is
        compatibility: str        # 1 sentence: why they fit (or don't)
        flags_explained: list[str]  # one entry per flag, empty list if no flags
        confidence: Literal["high", "medium", "low"]
        suggested_action: str | None  # populated only if confidence is medium/low

    class GroupExplanation(BaseModel):
        """Full explanation record attached to a group_id."""
        group_id: str
        summary: str
        compatibility: str
        flags_explained: list[str]
        confidence: Literal["high", "medium", "low"]
        suggested_action: str | None

    class OperatorOverride(BaseModel):
        """A single structured override to apply to the assignment."""
        pipeline_user_id: str
        from_group_id: str
        to_group_id: str

    class ReviewedResult(BaseModel):
        """Final output of the review workflow."""
        event_id: str
        groups: list[dict]            # serialised Group objects (model_dump())
        explanations: list[GroupExplanation]
        operator_notes: str
        overrides_applied: list[OperatorOverride]
        workflow_trace_id: str        # placeholder "local" in Phase 03; LangSmith ID in 03-02
        reviewed_at: str              # ISO 8601
    ```

    Avoid: adding validator logic here — keep schemas as pure data containers.

  <name>Task 4: Implement LangGraph workflow (workflow.py)</name>
  <files>src/agent/workflow.py</files>
  <action>
    Implement the full LangGraph review workflow in `src/agent/workflow.py`.

    **AgentState (TypedDict):**
    ```python
    from typing import TypedDict, Annotated
    import operator

    class AgentState(TypedDict):
        assignment: GroupAssignment           # input
        feature_vectors: list[UserFeatureVector]  # input
        event_type: str                       # input: "singles" | "social"
        group_explanations: list[GroupExplanation]   # set by explain_groups
        requires_human: bool                  # set by flag_review
        flagged_groups: list[str]             # group_ids, set by flag_review
        operator_notes: str                   # set by human_checkpoint
        overrides: list[OperatorOverride]     # set by human_checkpoint
        reviewed_result: ReviewedResult | None  # set by compile_output
    ```

    **Node: `explain_groups` (async):**
    - Build fv_map from feature_vectors
    - For each group in assignment.groups, construct the USER_PROMPT_TEMPLATE string
      (fit_label = "great" if cohesion >= 0.68, "okay" if >= 0.42, else "poor")
    - Use `ChatAnthropic(model="claude-haiku-4-5-20251001", max_tokens=400)` with
      `.with_structured_output(GroupExplanationSchema)`
    - Dispatch all LLM calls concurrently: `results = await asyncio.gather(*coroutines)`
      where each coroutine is `await llm_structured.ainvoke([SystemMessage(SYSTEM_PROMPT), HumanMessage(user_prompt)])`
    - Convert each result + group_id into a GroupExplanation object
    - Return state update: `{"group_explanations": [...]}`

    **Node: `flag_review` (sync):**
    Examine all GroupExplanation objects and all Group objects. Set requires_human = True if ANY of:
    1. Any explanation has `confidence == "low"`
    2. Any Group in assignment.groups has `fit_color == "#ef4444"` (poor cohesion)
    3. More than 25% of groups have `confidence == "medium"`
    Collect flagged group_ids from conditions 1 and 2.
    Return `{"requires_human": bool, "flagged_groups": [...]}`

    **Node: `human_checkpoint`:**
    Use LangGraph's `interrupt()` function to pause the workflow.
    The interrupt value is a dict with `flagged_groups` and `explanations` for the operator.
    After resume, the caller provides `{"operator_notes": str, "overrides": list[OperatorOverride]}`.
    Return the operator input as state update.

    **Node: `compile_output` (sync):**
    Apply any `overrides` from state using `apply_override()` from matching module.
    Build fv_map, partner_map (empty partner_map for now — override validation is the caller's
    responsibility at the API layer). Apply each override in sequence; catch ConstraintError and
    log a warning (do not raise — the operator explicitly chose this).
    Assemble ReviewedResult with:
    - `event_id` from assignment
    - `groups` = [g.model_dump() for g in final_assignment.groups]
    - `explanations` from state
    - `operator_notes` from state (default "")
    - `overrides_applied` from state (default [])
    - `workflow_trace_id` = "local" (LangSmith ID added in Plan 03-02)
    - `reviewed_at` = datetime.now(UTC).isoformat()
    Return `{"reviewed_result": reviewed_result}`

    **Graph assembly:**
    ```python
    from langgraph.graph import StateGraph, END
    from langgraph.checkpoint.memory import MemorySaver  # for interrupt support

    builder = StateGraph(AgentState)
    builder.add_node("explain_groups", explain_groups)
    builder.add_node("flag_review", flag_review)
    builder.add_node("human_checkpoint", human_checkpoint)
    builder.add_node("compile_output", compile_output)

    builder.set_entry_point("explain_groups")
    builder.add_edge("explain_groups", "flag_review")
    builder.add_conditional_edges(
        "flag_review",
        lambda s: "human_checkpoint" if s["requires_human"] else "compile_output",
    )
    builder.add_edge("human_checkpoint", "compile_output")
    builder.add_edge("compile_output", END)

    checkpointer = MemorySaver()
    graph = builder.compile(checkpointer=checkpointer, interrupt_before=["human_checkpoint"])
    ```

    **Public function:**
    ```python
    async def run_review_workflow(
        assignment: GroupAssignment,
        feature_vectors: list[UserFeatureVector],
        event_type: str = "social",
        thread_id: str | None = None,
    ) -> ReviewedResult:
        """Run the full review workflow. Returns ReviewedResult on clean path.
        Raises GraphInterrupt on flagged path — caller must handle and resume."""
        import uuid
        tid = thread_id or str(uuid.uuid4())
        config = {"configurable": {"thread_id": tid}}
        initial_state: AgentState = {
            "assignment": assignment,
            "feature_vectors": feature_vectors,
            "event_type": event_type,
            "group_explanations": [],
            "requires_human": False,
            "flagged_groups": [],
            "operator_notes": "",
            "overrides": [],
            "reviewed_result": None,
        }
        final = await graph.ainvoke(initial_state, config=config)
        return final["reviewed_result"]
    ```

    Avoid:
    - Recompiling the graph on every call — `graph` is module-level
    - Storing real user IDs or names in any log message
    - Using synchronous LLM calls inside an async node
