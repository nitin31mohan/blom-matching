<objective>
## Goal
Add NL override parsing and LangSmith trace ID propagation to complete the Phase 03 agent layer:
(1) `parse_operator_overrides()` — LLM call that converts free-text operator notes into structured `list[OperatorOverride]`, with invalid IDs filtered before returning; (2) `compile_output` calls the parser when `operator_notes` is non-empty; (3) `run_review_workflow` is decorated with `@traceable` and captures the LangSmith run ID into `ReviewedResult.workflow_trace_id`.

## Purpose
Closes the human-in-the-loop loop end-to-end: operator types a plain-English instruction ("Move user abc from group-01 to group-02"), the system parses it into a structured override and applies it. LangSmith tracing gives the Blom founder a full audit trail of every review session — who approved what and why.

## Output
- `src/agent/override_parser.py` — `OverrideParseResult` schema + `parse_operator_overrides()` async function
- `src/agent/workflow.py` — `compile_output` made async + NL parser call + `run_review_workflow` decorated with `@traceable` + `workflow_trace_id` wired from `get_current_run_tree()`
- `src/agent/__init__.py` — `parse_operator_overrides` added to public exports
- `tests/agent/test_override_parser.py` — AC-4 covered with mocked LLM
- `tests/agent/test_workflow.py` — AC-5 added (trace ID wired correctly)
</objective>

  <name>Task 1: Implement NL override parser (override_parser.py)</name>
  <files>src/agent/override_parser.py</files>
  <action>
    Implement `src/agent/override_parser.py` with:

    **Schema (Pydantic v2):**
    ```python
    class OverrideParseResult(BaseModel):
        overrides: list[OperatorOverride]
        parse_confidence: Literal["high", "medium", "low"]
        unparsed_intent: str | None  # any instruction the LLM could not parse
    ```

    **Prompt constants (module-level strings):**
    ```
    OVERRIDE_SYSTEM_PROMPT = """You are a structured data extractor. The user will
    give you plain-English instructions for moving attendees between groups.
    Extract each move as a structured override. Only output moves that reference
    valid user IDs and group IDs from the context provided.
    If an instruction is ambiguous or references unknown IDs, set unparsed_intent."""

    OVERRIDE_USER_TEMPLATE = """Assignment context:
    {group_context}

    Operator instructions:
    {operator_notes}

    Extract all valid move instructions as OperatorOverride objects.
    Ignore any instruction referencing IDs not listed above."""
    ```

    **Helper `_build_group_context(assignment: GroupAssignment) -> str`:**
    Produces a compact plain-text summary of each group and its members:
    ```
    group-01: [uid1, uid2, uid3]
    group-02: [uid4, uid5]
    ```
    (One line per group, user_ids separated by commas.)

    **Public async function:**
    ```python
    async def parse_operator_overrides(
        notes: str,
        assignment: GroupAssignment,
    ) -> list[OperatorOverride]:
        """Parse free-text operator notes into validated OperatorOverride objects.

        LLM extracts structured overrides; any override referencing a user_id or
        group_id not present in `assignment` is silently dropped.
        Returns empty list if notes is blank or LLM extracts nothing valid.
        """
    ```

    Implementation:
    - If `notes.strip()` is empty, return `[]` immediately (no LLM call)
    - Build `group_context` string and format `OVERRIDE_USER_TEMPLATE`
    - Use `ChatAnthropic(model="claude-haiku-4-5-20251001", max_tokens=300)` with
      `.with_structured_output(OverrideParseResult)`
    - Call `.ainvoke([SystemMessage(OVERRIDE_SYSTEM_PROMPT), HumanMessage(user_prompt)])`
    - Validate: build `valid_user_ids` = all user_ids across all groups; `valid_group_ids` = set of group_ids
    - Filter: keep only overrides where `pipeline_user_id in valid_user_ids`,
      `from_group_id in valid_group_ids`, `to_group_id in valid_group_ids`,
      and `from_group_id != to_group_id`
    - Log a warning for any filtered override (do not raise)
    - Return the filtered list

    Import: `from src.agent.schemas import OperatorOverride`
    Import: `from src.matching.assignment import GroupAssignment`

    Avoid: raising exceptions on bad LLM output — filter silently.
    Avoid: logging user_ids or group content in warning messages (PII risk if real data).
  </action>

  <name>Task 2: Wire parser + LangSmith into workflow.py</name>
  <files>src/agent/workflow.py</files>
  <action>
    Make two targeted changes to `src/agent/workflow.py`:

    **Change A: Make `compile_output` async and call the NL parser**

    Convert `def compile_output(state)` to `async def compile_output(state)`.

    At the top of `compile_output`, after extracting `operator_notes` and `explicit_overrides`:
    ```python
    from src.agent.override_parser import parse_operator_overrides

    operator_notes = state.get("operator_notes") or ""
    explicit_overrides: list[OperatorOverride] = state.get("overrides") or []

    parsed_overrides: list[OperatorOverride] = []
    if operator_notes.strip():
        parsed_overrides = await parse_operator_overrides(operator_notes, assignment)

    all_overrides = explicit_overrides + parsed_overrides
    ```
    Use `all_overrides` (not `overrides`) everywhere overrides are applied downstream.

    Note: LangGraph supports async nodes — no graph recompilation needed, just the function signature change.

    **Change B: Add `@traceable` and capture LangSmith run ID in `run_review_workflow`**

    Add import: `from langsmith import traceable, get_current_run_tree`

    Decorate `run_review_workflow` with `@traceable(name="blom_review_workflow")`.

    After `final = await graph.ainvoke(initial_state, config=config)`:
    ```python
    result: ReviewedResult = final["reviewed_result"]

    # Capture LangSmith run ID if tracing is active
    run_tree = get_current_run_tree()
    if run_tree is not None:
        result = result.model_copy(update={"workflow_trace_id": str(run_tree.id)})

    return result
    ```
    (Replace the bare `return final["reviewed_result"]` line.)

    Avoid:
    - Changing the graph topology or recompiling `graph`
    - Importing `parse_operator_overrides` at module level (import inside function to avoid circular)
    - Catching exceptions from `get_current_run_tree()` — it should not raise
  </action>
