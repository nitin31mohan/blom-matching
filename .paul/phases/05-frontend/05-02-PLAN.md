<objective>
## Goal
Wire the operator tool into an interactive two-pane UI: Zustand store tracks selection state; clicking an attendee node populates a sidebar showing AttendeeDetail and GroupSummary panels.

## Purpose
Transforms the static canvas from Plan 05-01 into the actual operator workspace — the human reviewer can click any attendee node and see their name, group, cohesion score, and flags without leaving the canvas. This is the core "human-in-the-loop" interaction surface.

## Output
- `canvas.store.ts` — Zustand store with selectedAttendeeId + actions
- `AttendeeDetail.tsx` — sidebar panel for selected attendee
- `GroupSummary.tsx` — sidebar panel for selected attendee's group
- Updated `AttendeeNode.tsx` — adds onClick with drag-safety guard
- Updated `ForceCanvas.tsx` — accepts `onNodeClick` prop, passes to nodes, handles SVG background click to clear selection
- Updated `App.tsx` — two-column layout (canvas | 320px sidebar), connects store, still uses mock data (API wiring deferred to Phase 06)
</objective>

    Implement the Zustand store for canvas selection state.

    ```ts
    import { create } from 'zustand'

    interface CanvasState {
      selectedAttendeeId: string | null
      selectAttendee: (id: string) => void
      clearSelection: () => void
    }

    export const useCanvasStore = create<CanvasState>((set) => ({
      selectedAttendeeId: null,
      selectAttendee: (id) => set({ selectedAttendeeId: id }),
      clearSelection: () => set({ selectedAttendeeId: null }),
    }))
    ```

    No additional state needed — selected group is always derived from the
    selected attendee (look up attendee.group_id in the attendees array).
    Avoid adding selectedGroupId as a separate field — it creates sync bugs.
