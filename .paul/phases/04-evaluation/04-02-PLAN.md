<objective>
## Goal
Implement the post-event feedback ingestion loop: collect per-attendee satisfaction ratings, compute group-level satisfaction scores, measure how well cohesion rank predicts satisfaction rank (Spearman correlation), and produce deterministic weight adjustment suggestions when correlation is low.

## Purpose
Closes the feedback loop for the Blom founder: after each event, attendees rate their experience. If the algorithm's cohesion ordering matched satisfaction ordering (high correlation), weights are confirmed. If not, the system suggests nudging weights toward uniform — making the next event's matching more balanced. Gives the founder a quantitative signal to tune the algorithm without manual trial and error.

- `src/evaluation/feedback.py` — `AttendeeRating`, `EventFeedback` Pydantic models + `group_satisfaction_scores()`, `cohesion_satisfaction_correlation()`, `suggest_weight_adjustments()` functions

  <action>
    Implement `src/evaluation/feedback.py` with:

    **Schemas (Pydantic v2):**
    ```python
    class AttendeeRating(BaseModel):
        pipeline_user_id: str
        group_id: str
        satisfaction: int   # 1–5 inclusive

        @field_validator("satisfaction")
        @classmethod
        def _validate_satisfaction(cls, v: int) -> int:
            if not 1 <= v <= 5:
                raise ValueError(f"satisfaction must be 1–5, got {v}")
            return v

    class EventFeedback(BaseModel):
        event_id: str
        ratings: list[AttendeeRating]
        collected_at: str   # ISO 8601
    ```

    **`group_satisfaction_scores(feedback: EventFeedback) -> dict[str, float]`:**
    - Group ratings by `rating.group_id`
    - For each group_id, compute `mean(r.satisfaction for r in group_ratings)`
    - Return `{group_id: mean_satisfaction}` — only groups that have at least one rating

    **`cohesion_satisfaction_correlation(assignment: GroupAssignment, feedback: EventFeedback) -> float`:**
    - Compute `sat_scores = group_satisfaction_scores(feedback)`
    - Find groups present in both `assignment.groups` and `sat_scores`
    - If fewer than 2 overlapping groups: return 0.0
    - Build two lists (same order): `cohesion_values` and `satisfaction_values`
    - Compute Spearman rank correlation:
      - Rank each list (1 = highest) using `scipy.stats.spearmanr`
      - Return `float(spearmanr(cohesion_values, satisfaction_values).statistic)`
    - Import: `from scipy.stats import spearmanr`

    **`suggest_weight_adjustments(current_weights: dict[str, float], rank_corr: float, delta: float = 0.05) -> dict[str, float]`:**
    - If `rank_corr >= 0.5`: return `dict(current_weights)` unchanged
    - If `rank_corr < 0.5`:
      - Find `max_key = max(current_weights, key=current_weights.get)`
      - Find `min_key = min(current_weights, key=current_weights.get)`
      - If `max_key == min_key` (all weights equal): return `dict(current_weights)` unchanged
      - Build new dict: decrease `max_key` by delta, increase `min_key` by delta, keep others
      - Clamp all values to [0.5, 2.0]: `max(0.5, min(2.0, v))`
      - Return the clamped dict

    Imports needed:
    - `from pydantic import BaseModel, field_validator`
    - `from scipy.stats import spearmanr`
    - `from src.matching.assignment import GroupAssignment`

    Avoid:
    - Raising if a group_id in feedback is not in the assignment (silently skip)
    - Modifying `current_weights` in place — always return a new dict
  </action>
