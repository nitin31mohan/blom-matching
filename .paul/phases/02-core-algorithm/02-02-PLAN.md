<acceptance_criteria>

## AC-1: All users assigned
```gherkin
Given an AffinityMatrix for N attendees
When assign_groups() is called
Then sum of all group sizes equals N
And GroupAssignment.unassigned is empty
```

## AC-2: Group size within bounds
```gherkin
Given config with group_size_min=3, group_size_max=6
When assign_groups() is called on an event with ≥6 attendees
Then every group has between 3 and 6 members (inclusive)
```

## AC-3: Friend pairs co-assigned
```gherkin
Given two attendees sharing the same friend_pair_id UUID
When assign_groups() is called with friend_pair_ids mapping both to the shared UUID
Then both attendees appear in the same Group in the returned GroupAssignment
```

## AC-4: Greedy improves cohesion
```gherkin
Given an affinity matrix with two clear sub-groups
  (users 0–3 identical profiles, users 4–7 polar-opposite profiles)
When assign_groups() is called with target_group_size=4
Then mean group_cohesion_score across groups is >= 0.7
  (greedy assignment keeps similar users together)
```

## AC-5: apply_override moves user
```gherkin
Given a GroupAssignment and a user with no friend pair
When apply_override() is called to move that user to another group
Then a new GroupAssignment is returned with the user in the target group
And cohesion_score for both affected groups is recomputed
```

## AC-6: apply_override rejects split pairs
```gherkin
Given a GroupAssignment where user A and their partner B are in group-01
When apply_override() is called to move only user A to group-02
Then ConstraintError is raised
```

    **`assign_groups(affinity, feature_vectors, friend_pair_ids, config=None) -> GroupAssignment`**

    Algorithm — step by step:

    1. Load config: read `assignment.group_size_min` (default 3), `assignment.group_size_max` (default 6), `assignment.target_group_size` (default 5)
    2. `N = len(affinity.user_ids)`. Raise ValueError if `len(feature_vectors) != N`.
    3. `K = max(1, ceil(N / target_group_size))` — initial group count
    4. Build `partner_map = build_friend_pair_map(list(affinity.user_ids), friend_pair_ids)`
    5. Initialize K mutable groups: `groups: list[list[str]] = [[] for _ in range(K)]`
    6. Track `assigned: set[str] = set()`

    **Step 6a — Pre-assign friend pairs (hard constraint):**
    - Collect unique pairs: `seen_pairs: set[frozenset]` to dedup
    - Iterate `partner_map.items()`. For each `(uid, partner)` where `frozenset({uid, partner})` not yet seen:
      - Add to seen_pairs
      - Find target group: round-robin by `pair_index % K`
      - If that group is already at `group_size_max`, try next groups in order until one has room, else append a new group
      - Add both uid and partner to the chosen group and to `assigned`
      - Increment pair_index

    **Step 6b — Greedy assignment of remaining users:**
    - `unassigned = [uid for uid in affinity.user_ids if uid not in assigned]`
    - Sort unassigned by `max(affinity.matrix[row_for_uid, :])` descending (most "matchable" first, tie-break by original order)
    - For each uid in sorted unassigned:
      - Find all groups where `len(group) < group_size_max`
      - If any open groups:
        - Compute `marginal_cohesion(affinity, uid, group_members)` for each open group (use `marginal_cohesion()` from similarity.py; returns 0.0 if group is empty)
        - Place uid in the group with highest marginal_cohesion (ties: pick least-full group)
      - If no open groups (overflow):
        - Place in least-full group regardless of max_size (forced overflow)
      - Add uid to `assigned`

    7. Build `fv_map: dict[str, UserFeatureVector] = {fv.user_id: fv for fv in feature_vectors}` for flag lookup
    8. For each group list, build a `Group` object:
       - `group_id = f"group-{i+1:02d}"`
       - `cohesion_score = group_cohesion_score(affinity, member_ids)`
       - `fit_color = group_fit_color(cohesion_score)`
       - `flags`: check `fv_map[uid].flags` for each member; if any has `"high_anxiety"`, add `"high_anxiety_present"` to group flags; if `len(members) < group_size_min`, add `"size_warning"`
    9. `assigned_at = datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")`
    10. Return `GroupAssignment(event_id=affinity.event_id, groups=tuple(groups), assigned_at=assigned_at, unassigned=())`

    **`apply_override(assignment, affinity, feature_vectors, move_user_id, from_group_id, to_group_id, partner_map, config=None) -> GroupAssignment`**

    1. Raise `ValueError` if `move_user_id` not in any group in `assignment`
    2. Raise `ValueError` if `from_group_id` or `to_group_id` not found in `assignment`
    3. Constraint check: if `move_user_id` in `partner_map`:
       - `partner = partner_map[move_user_id]`
       - Find group containing `partner`. If partner is in `from_group_id` (moving would split the pair): raise `ConstraintError(f"Cannot split friend pair: {move_user_id!r} and partner both in {from_group_id!r}")`
    4. Rebuild groups: copy all Group objects; for the two affected groups, rebuild their user_ids with the move applied
    5. Recompute `cohesion_score` + `fit_color` + `flags` for both affected groups only
    6. Return new `GroupAssignment` with updated groups, same `event_id` and `assigned_at`

    Dependencies: numpy, datetime (stdlib), pydantic, math.ceil. No new packages needed.
    Import: `from src.matching.similarity import group_cohesion_score, group_fit_color, marginal_cohesion`
    Import: `from src.matching.constraints import build_friend_pair_map, ConstraintError`
