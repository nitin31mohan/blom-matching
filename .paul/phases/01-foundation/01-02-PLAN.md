<objective>
## Goal
Implement the PII anonymisation module: strip real identifiers at the pipeline boundary, maintain an in-memory reverse mapping for DB write-back, and produce fully untraceablee demo exports.

## Purpose
Real PII must never travel through the matching pipeline. This module is the single enforcement point — called immediately when operator tool receives data from Blom's backend, before any feature engineering, logging, or LangGraph invocation. The demo prong depends on export_for_demo() to produce synthetic exports that sever all links to real users.

## Output
- `src/data/anonymiser.py` — AnonymisedAttendee model + strip_pii(), build_reverse_mapping(), export_for_demo()
- `tests/data/test_anonymiser.py` — unit tests covering all four skill ACs

## AC-1: PII stripped at boundary
```gherkin
Given a raw attendee dict containing id, name, email, phone, and all 20 quiz fields
When strip_pii() is called
Then the returned AnonymisedAttendee has no name, email, or phone field
And pipeline_user_id is a valid UUID4 string not equal to the original id
And display_name contains only the first token of name (no surname)
And quiz_responses contains exactly all 20 quiz fields, values unchanged
```

## AC-2: Reverse mapping correctness
```gherkin
Given a list of 10 raw attendees passed through strip_pii()
When build_reverse_mapping(stripped_list, original_list) is called
Then the returned dict has exactly 10 entries
And each pipeline_user_id key maps to the correct original id value
```

## AC-3: Demo export untraceability
```gherkin
Given the same AnonymisedAttendee passed through export_for_demo() twice with different seeds
When the two outputs are compared
Then display_name differs between the two outputs
And religious_identity differs between the two outputs
And country differs between the two outputs
And pipeline_user_id differs between the two outputs
```

## AC-4: No PII in demo export
```gherkin
Given a batch of 50 attendees processed through export_for_demo()
When each record is scanned for email patterns (@) and UK mobile patterns (07\d{9})
Then no matches are found
```

## AC-5: Missing name fallback
```gherkin
Given a raw attendee dict where the name field is absent or blank
When strip_pii() is called
Then display_name is "Attendee" followed by a 4-digit random suffix
```

    ```
    pipeline_user_id: str   # Fresh UUID4 — used throughout pipeline
    display_name: str       # First name only
    quiz_responses: dict    # All 20 quiz fields, unchanged values
    ```

    **QUIZ_FIELDS** — module-level constant listing the 20 quiz field names
    (copy from SKILL-feature-engineering.md schema table — these are the keys
    that must be present in quiz_responses):
    gender, industry, country, energised_meeting_people,
    keeps_atmosphere_harmonious, enjoys_unfamiliar_experiences, shows_up_on_time,
    anxious_in_social_situations, interested_in_current_events, religious_identity,

    shares_personal_stories, weekend_energy_level, preferred_activity_time, humour_style

    **strip_pii(raw_attendee: dict) -> AnonymisedAttendee**:
    - Generate pipeline_user_id = str(uuid.uuid4()) — never derived from any input
    - Extract display_name:
        - raw_name = raw_attendee.get("name", "").strip()
        - If raw_name is blank: display_name = "Attendee" + str(random 4-digit suffix)
          Use secrets.randbelow(9000) + 1000 for the suffix (no seed dependency)
        - Else: display_name = raw_name.split()[0]
    - Build quiz_responses by extracting only QUIZ_FIELDS keys from raw_attendee.
      If a key is absent, set it to None. Do NOT include id, name, friend_pair_id,
      or any other non-quiz key.
    - All other fields not in QUIZ_FIELDS are silently dropped — no logging.

    **build_reverse_mapping(
        stripped_list: list[AnonymisedAttendee],
        original_list: list[dict],
    ) -> dict[str, str]**:
    - Return {a.pipeline_user_id: orig["id"] for a, orig in zip(stripped_list, original_list)}
    - This dict is NEVER written to disk. Add a docstring warning: "In-memory only. Never persist or log."

    Avoid: importing anything other than uuid, secrets, and pydantic. No numpy, no faker.
  </action>

       a. Likert perturbation: for each of the 12 Likert fields, with 40% probability

          the same region bucket (see COUNTRY_REGIONS below). If country is None
